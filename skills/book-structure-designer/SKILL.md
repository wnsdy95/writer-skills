# Book Structure Designer

## Workflow

Turn a book idea or research base into a coherent structure. Start by defining the book's promise, reader, central question, scope, and organizing principle.

Read:

- `references/book-concept-brief.md` for the concept brief
- `references/structure-patterns.md` for book type structures
- `references/chapter-card-system.md` for chapter-level planning

## Structure Rules

- A table of contents is not a list of topics; each chapter must do a job.
- Practical books should move from reader problem to transformation.
- Scholarly/technical books should move from definitions to argument/system to application.
- Narrative nonfiction should balance chronology, evidence, and narrative momentum.
- Fiction structure belongs in `fiction-book-writing` unless the user only needs high-level table-of-contents style planning.

## Output Contract

End with:

```markdown
## Book Concept Brief

- Working title:

- Reader promise:

## Table of Contents

## Chapter Card

- Key claims/scenes:
- Examples/evidence:
- Open risks:

## Structure Rationale
```
