# Setup and Payoff

## Types

- `foreshadowing`
- `clue`
- `red_herring`
- `symbol`
- `rule`
- `object`
- `dialogue`
- `wound`
- `relationship`
- `image_pattern`

## Definitions

- setup: a planted element that creates expectation, possibility, clue, rule, object, symbolic pattern, or emotional promise
- payoff: a later moment that uses the setup to create satisfaction, surprise, dread, revelation, or inevitability
- red herring: a true visible detail that supports a false but fair interpretation
- orphan setup: a planted element with no remaining payoff
- unsupported payoff: a reveal, solution, or emotional turn that has not been earned

## Rules

- Major payoff requires prior setup.
- Conspicuous setup needs payoff, conversion to red herring, or removal.
- Mystery clues must feel fair in retrospect.
- A twist should be surprising before the reveal and inevitable after it.
- Do not solve the central conflict by accident, unexplained intuition, or a rule first introduced at the climax.

## Fair-Play Clue Matrix

Use this for mystery, thriller, court intrigue, investigation, conspiracy, and twist-heavy fiction.

| field | required question |
|---|---|
| true_meaning | What does the clue actually prove? |
| visible_form | What exact detail appears on the page? |
| misleading_inference | What false conclusion can the reader reasonably draw? |
| reader_access | Has the reader seen the clue before the reveal? |
| detective_access | Does the investigating character know the same information? |
| solution_dependency | Is this required, supporting, thematic, or decorative? |
| fairness_test | Can a rereader point to the clue without needing hidden off-page evidence? |

## Revision Actions

- Plant earlier when payoff is unsupported.
- Reinforce when a setup is too invisible.
- Hide better when a clue is too obvious.
- Reclassify as red herring when the visible fact remains useful but the inference changes.
- Cut when setup creates a promise the story no longer intends to satisfy.
