# Book Project Architect

## Workflow

Classify the user's book idea before giving writing advice. Always determine:

- market classification
- knowledge domain
- manuscript form
- reader
- evidence burden
- research methods
- risk level
- publishing intent
- current project stage

Read `references/project-diagnosis.md` for the diagnostic checklist. Read `references/taxonomy-router.md` when the book type or skill route is unclear. Read `references/risk-routing.md` if the project touches real people, health, law, finance, crime, politics, vulnerable groups, or sensitive history.

## Routing

- Use `book-research-designer` when the next question is what to research or verify.
- Use `book-structure-designer` when the next question is concept, argument, table of contents, or chapter architecture.
- Use `book-drafting-editor` when the next question is drafting, revision, or manuscript improvement.
- Use `fiction-book-writing` when invented narrative is central.
- Use `fiction-story-bible-manager` when a fiction project needs canon, terms, timelines, setup/payoff, or continuity memory.
- Use `narrative-nonfiction-writing` when real people, memory, events, archive, interviews, or reportage are central.
- Use `practical-nonfiction-writing` when reader transformation, method, framework, exercises, or checklists are central.
- Use `scholarly-technical-writing` when expertise, research, definitions, citations, code, formulas, or technical explanation dominate.
- Use `visual-hybrid-book-writing` when page, image, spread, caption, panel, recipe, or visual sequence is structural.
- Use `high-stakes-book-review` for health, law, finance, crime, politics, social conflict, named people, vulnerable groups, sensitive history, or high-harm claims.
