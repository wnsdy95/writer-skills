# Contributing

Contributions are welcome when they improve the clarity, safety, or usefulness of the writing workflows in this repository.

## Good Contributions

- Fix broken skill references, missing assets, or validation problems.
- Improve output contracts so skills produce more concrete deliverables.
- Add source-quality, risk-review, or continuity checks that reduce reader harm.
- Clarify documentation without making the skills more verbose than needed.

## Before Opening a Pull Request

1. Keep each change focused.
2. Run `scripts/verify-writer-collections.sh`.
3. Do not add private notes, unpublished client material, secrets, or copyrighted text that you do not have permission to license under MIT.
4. For medical, legal, financial, real-person, crime, or sensitive-history material, explain the evidence basis and risk model in the pull request.

## Style

- Keep skill instructions direct and operational.
- Prefer named outputs, ledgers, checklists, and decision records over general advice.
- Keep long reference material in `references/` rather than overloading `SKILL.md`.
- Use stable file paths when a skill refers to resources.
