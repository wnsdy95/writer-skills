# Security Policy

## Reporting

Please report security issues privately through GitHub's private vulnerability reporting if it is available for this repository. If it is not available, open an issue with only non-sensitive details and ask for a private contact path.

Do not publish secrets, private user data, client manuscripts, unpublished source material, or exploit details in public issues or pull requests.

## Scope

This repository mainly contains Markdown skills, templates, and validation scripts. Relevant security issues include:

- leaked secrets or credentials
- private or identifying information accidentally included in examples
- unsafe instructions that could cause legal, medical, financial, or privacy harm
- supply-chain risks in scripts or automation

## Supported Branches

The `main` branch is the supported branch.
